
# Описание UI 

Сделай мне вэб-панель, похожую на панель 3x-ui (https://github.com/MHSanaei/3x-ui) для ядра sing box (https://github.com/SagerNet/sing-box) в стиле chatgpt openai 

## Стэк 

архитектура **SPA (Single Page Application) + Микро-бэкенд**.
Vite + React + TypeScript + Tailwind CSS + Framer Motion + Recharts
Сделать адаптивный дизайн, панель будет использоваться всеми устройствами. 

## Основные визуальные параметры:

1. Основной фон: глубокий темно-серый/угольный (`#171717` — как в темной теме ChatGPT). Никакого абсолютно черного цвета, интерфейс должен дышать.
2. Контейнеры или карточки: Очищенный минимализм. Фон `#212121` или `#2f2f2f` для выделяющихся элементов. Едва заметные, мягкие скругления углов (`rounded-xl` / `12px`), тонкие границы (`1px solid rgba(255,255,255,0.08)`) и мягкое, едва уловимое размытие заднего плана.
3. Акцентные цвета, которые стоит использовать экономно: кнопки действий или активные состояния фирменный зеленый ChatGPT (`#10a37f`) или чистый белый (`#ffffff`) на темном фоне; мягкий приглушенный серый (`#b4b4b4`) для вторичных элементов и ссылок; Ошибки / Остановлено: Сдержанный пастельно-красный (`#ef4444`); статусы (Успех / Онлайн): Интеллектуальный мягкий зеленый (`#19c37d`). 
4. Основной шрифт: Inter. Все заголовки стандартным регистром. Sentence case или Title Case. Никакого ALL-CAPS и моноширинного JetBrains Mono в интерфейсе, **кроме** строго технических данных: токенов, UUID, логов и IP-адресов.

## Модальные окна и формы 

Все модальные окна (создание инбаунда, редактирование клиента, отображение QR) должны следовать паттерну: 
1. Плавное, мягкое масштабирование из центра (`scale(0.95) -> scale(1)`) одновременно с деликатным нарастанием прозрачности (`opacity: 0 -> 1`). Настройки Framer Motion используют физику «затухания» (_ease-out_, длительность `0.2s`), без избыточной прыгучести пружин.
2. Фон страницы при модальном окне: Полупрозрачный темный оверлей (`rgba(0, 0, 0, 0.7)`) с глубоким, премиальным размытием (`backdrop-filter: blur(8px)`).
3. Поля ввода (Inputs): Фон `#2f2f2f`, тонкая серая граница. При фокусе (клике) граница становится чисто белой или мягко подсвечивается зеленым ChatGPT акцентом (`#10a37f`), без неонового свечения и bloom-эффектов. Выпадающие списки открываются мягким скольжением вниз с легким затуханием.
4. Cелекторы: выпадающее меню в стиле выпадающих списков выбора моделей (GPT-4o/GPT-3.5). Мягкое появление списка, при наведении на элемент — аккуратный серый фон.
5. Валидация ошибок: при некорректном вводе рамка инпута становится сдержанно-красной (`#ef4444`). Поле не вибрирует и не трясется. Ниже поля появляется аккуратный лаконичный текст ошибки красного цвета, напечатанный стандартным шрифтом. 


## Микро-взаимодействия и анимации 

Все анимации плавные, текучие и ощущения высокотехнологичности 

1. Переходы между страницами, навигация между пунктами боковой панели (сделать боковую панель как такое меню, которые скрывается, если уводишь с него курсор и появляется если наводишь в эту зону для пк, а можно зафиксировать и сделать её всегда открытой). Контент страниц и пункты меню переключаются через аккуратный `fade-in` (проявление) длительностью `150ms`. Комбинируется с едва заметным подъемом вверх (всего на `4px`), что создает ощущение плавного проявления интерфейса, а не его агрессивного движения. 
2. Наведение курсора на любой интерактивный элемент (карточки, кнопки, пункты меню): его фон становится на несколько тонов светлее (например, переход от `#212121` к `#2a2a2a`), либо плавно увеличивается яркость текста. Всё происходит за `200ms` с мягким сглаживанием.
3. Состояние клика, нажатие на основную кнопку (например, "Добавить инбаунд"):  При нажатии на основные кнопки элемент слегка и мгновенно уменьшается в масштабе (`scale(0.98)`), имитируя физический упругий клик. Никаких ripple-эффектов (волн) и пульсаций. 
4. Обновление данных, динамическое изменение метрик (цп, трафик): Обновление графиков и цифр (ЦП, трафик) происходит через плавную интерполяцию значений. Цифры не прыгают и не крутятся как в матрице — они аккуратно и быстро меняются через стандартный `cross-fade` (одно значение растворяется, другое проявляется).
   
## Описание страниц 

### Основной макет 

1. Sidebar: Выполнена в стиле левой панели ChatGPT. Самый темный элемент интерфейса (`#171717`). Вверху — лаконичный текстовый логотип или минималистичная иконка. Пункты меню: _Dashboard, Inbounds, Clients, Settings, Logs_. Текст написан обычным шрифтом Inter, стандартным регистром (не капсом). Активный пункт выделяется мягким светло-серым/белым фоном скругленной формы (`bg-[#212121]`), дублируя логику активного чата. Панель статична (всегда открыта для ПК) для удобства работы.
2. Main content: Фон `#212121` (чуть светлее сайдбара). Карточки лежат в одной плоскости с фоном, выделяясь за счет более светлого тона (`#2f2f2f`) или тонких разделительных линий.
3. Top bar: Справа аккуратная текстовая плашка "Sing-Box: Active" с аккуратной статичной зеленой точкой (`#19c37d`) без пульсаций.

### Страница Dashboard 

Это хаб визуализации системных и основных статистик. Ниже блоки, которые нужно расположить в порядке их описания в тексте. 

#### Горизонтальный блок в самом верху

Оформлен, как единая экструдированная панель с матовым стеклянным эффектом glassmorphism. 
Содержит: 
1. статус ядра: элемент в виде аккуратной серой капсулы. внутри которой горит ровный зеленый (`Active`) или красный (`Stopped`) маркер-точка. 
2. аптайм сервера: метка выводятся аккуратными серыми блоками сбалансированного цвета. 
3. глобальный трафик: суммарные счетчики за всё время Total sent и Total received. 

#### Верхний ряд метрик 

4 парящие 3D карточки: карточки для ЦП, ОЗУ, Общего Трафика сети и Диска.

1. ЦП. Тонкие изящные сплошные линии прогресс баров в стиле современных UI-систем. 
2. ОЗУ: То же самое, что и ЦП, двойной индикатор. Верхняя полоса — оперативная память, нижняя (более тонкая) — Swap. 
3. Диск: сегментированная горизонтальная линия, показывающая занятое пространство на сервере.
4. Трафик дня / месяца: сплит-карточка. Показывает расход трафика за текущие сутки и за текущий месяц. 

#### Панель соединений и пользователей 

Две массивные карточки, отображающие сетевую активность:
1. Inbounds Active: Количество работающих портов/протоколов. К
2. Clients Telemetry: Двухуровневый счетчик: `Total users: 124` и индикатор `Online now: 18`. Количество онлайн пользователей будет меняться по веб-сокетам. 

#### Центральный график скорости 

1. Интерактивный график: входящая скорость — циан, исходящая фиолетовый. 
2. График Recharts со сглаженными тонкими линиями. Входящая скорость — приглушенный изумрудно-зеленый, исходящая — нейтральный серый. При наведении появляется тонкая вертикальная линия, а в точке пересечения — аккуратный белый кружок. Подсказка (Tooltip) оформлена в виде чистого белого или темного мини-окна с аккуратными тенями.

#### Блоки для быстрого перехода 

Справа или сразу под графиком располагаются три широкие интерактивные кнопки-карточки для мгновенного перехода в ключевые разделы

Карточки-ссылки. При наведении на них иконка (сеть, пользователи, логи) не увеличивается, а становится ярче, фон карточки слегка светлеет, а текст плавно смещается вправо на `4px`. При клике страница плавно растворяется (`fade-out`), открывая нужный раздел.. 

Они выглядят как шлюзы в терминале, которые переносят тебя на: 
1. На страницу управления инбаундами. 
2. На страницу управление клиентами.
3. На страницу логов. 
   
### Страница inbounds 

Сетка или список прослушиваемых протоколов ядра (функциональность 3x-ui)

- Кнопка «Новая конфигурация» Оформлена как кнопка "New Chat" в ChatGPT. Стильный скругленный прямоугольник с белым фоном и черным текстом (или темно-серый с тонкой рамкой). При наведении слегка меняет прозрачность.

#### Карточка создания инбаунда 

Скругленный элегантный контейнер, фон `#2f2f2f`. Форма разделена на аккуратные раскрывающиеся текстовые секции (аккордеоны) с тонкими стрелочками.

##### Шапка карточки 

Строгий лаконичный заголовок "Создание входящего соединения". Крестик закрытия при наведении просто становится чуть ярче. Закрытие по ESC работает мгновенно.

##### Блок 1: основные параметры 

1. Поле примечание: Визуально прямоугольный инпут. Плейсхолдер: `e.g, vadim-vless#0001`.
2. Селектор протоколов. Кастомный выпадающий список, по умолчанию выбрано Naive Proxy. Доступные варианты: `Naive Proxy`, `VLESS`, Hysteria2`. 
3. Поле порт: числовой инпут рядом с селектором протокола. Справа внутри инпута находится иконка «игральной кости» (Randomize). При клике быстро вращается один раз без шлейфов, генерируя случайный свободный порт.
4. Поле Traffic с кастомным выпадающим списком: never (по умолчанию), hourly, daily, weekly, monyhly. 
5. Добавить выбор Transmission: TCP (RAW), mKCP, grpc, WebScoket, XHTTP, HTTPUpgrade. И к каждому из них свои настройки, которые есть в 3x-ui. 

##### Блок 2: транспорт и безопасность 

Содержимое этого блока динамически меняется в зависимости от выбранного протокола.

1. Sniffing. Интегрированный в sing-box аналог xray-сниффинга. Выглядит как футуристичный тумблер. Во включенном состоянии загорается кибер-зеленым (`#00FF66`), а под ним плавно появляются чекбоксы: `HTTP`, `TLS`, `QUIC`, `FAKEDNS`. Под ними свитчеры «Metadata only» и «Route only» и две плашки ввода IPs Excluded и Domains Excluded. 
2. TLS / Reality. Три кнопки-вкладки: `NONE`, `TLS`, `REALITY`. Выглядят как переключатель (Segmented Control) в настройках ChatGPT. Активная вкладка становится светлее и выделяется сплошным цветом, неактивные — остаются серыми. 
3. Динамические поля для Reality (появляются если выбран reality). Поля: dest, SNI, Short IDs. Блок генерации ключей (X25519 Private/Public Key): Две строки с ключами. Рядом кнопка, при нажатии на `[ Generate Keypair ]` старые UUID/ключи заменяются на новые мгновенно. 

##### Шаблон пользователя 

Позволяет сосздать первого клиента для этого инбаунда, как в 3x-ui 

- поле User ID 
- поле UUID, автоматически создается 
- поле Subscription, которое позволяет вставить определенное значение, чтобы пользователь подключался к одной подписке. 
- трафик и время. Два инпута: `Total Flow (GB)` и `Expiry Date`.
- кнопка свитчер, «Start After First Use», чтобы время пошло, после первого использования 

##### Нижняя панель управления 

Закрееплена в самом низу карточки, отделена от контента тонкой фиолетовойили циановой линией-разделителем. 

- Кнопка ОТМЕНА. Она становится красной, когда на нее наводишься. 
- Кнопка "Сохранить" — залита зеленым цветом `#10a37f`. При клике внутри нее появляется минималистичный круговой лоадер (Spinner), кнопка слегка сжимается. При успешном сохранении модальное окно плавно исчезает, а сверху страницы опускается аккуратная плашка-уведомление (Toast) в стиле ChatGPT: "Соединение успешно создано".

#### Список инбаундов 

1. Каждый инбаунд это отдельная аккуратная строка, плоская.  
2. Детали карточки: Протокол, Порт, Примечание, Статус (Вкл/Выкл).
3. Переключатель статуса (On/Off) — минималистичный тумблер (Toggle). В выключенном состоянии — серый, во включенном — плавно окрашивается в зеленый (`#10a37f`). 
4. Клик по карточке плавно разворачивает ее вниз, показывая текстовые кнопки действий: _Изменить, Клонировать, QR-код, Удалить_.

### Страница Client Manager 

Сетка пользователей и использование ресурсов. 

1. Таблица клиентов. Строгий, аккуратный интерфейс табличных данных. Простые строки с легким изменением фона при наведении (эффект `hover:bg-[#2f2f2f]`).
2. Столбцы: имя, использование данных (входящие и исходящие), дата истечения, инбаунд, статус. 
3. Полоса использования данных: Изящный тонкий progress bar. По мере заполнения он остается нейтрального серого или белого цвета, переходя в контрастный красный только при критическом приближении к лимиту (более 90%)
4. Есть фильтрация по столбцам, можно отфильтровать пользователь только по определенному инбаунду. 

При нажатии на клиента открывается модальное окно, показывающее детали подключения и возможность их редактирования, параметры для клиента бери из шаблона создания клиента выше. А внизу ещё кнопки получить qr и копировать ссылку. При клике на "Получить QR" открывается небольшое аккуратное модальное окно по центру. Сам QR-код гармонично вписан в светлую или темную подложку скругленной формы. Кнопка "Копировать ссылку" при нажатии меняет свой текст на "Скопировано!", а сверху дублируется мягкое системное уведомление.

### Страница Logs 

Экран, похожий на консоль. 
- Полноэкранная, текстовая область
- Текст: строгий моноширинный шрифт (системный SF Mono или Courier New) небольшого размера для удобства чтения. Цвет текста — мягкий беловато-серый. Критические ошибки подсвечиваются пастельно-красным.
- Критический эффект: Логи добавляются снизу. Новые строки появляются мгновенно, но с едва заметным, очень быстрым `fade-in` (`50ms`), чтобы непрерывный поток данных не утомлял глаза мерцанием.
- Верхняя панель фильтрации (светящиеся поля ввода) для поиска по логам. Расположена вверху, представляет собой лаконичную строку поиска в стиле поисковой строки ChatGPT, с аккуратной иконкой лупы внутри.
